The Line

4 stops from request to ticket
MILE 01

Request Arrives

A booking transaction is stamped with its start timestamp the moment it hits the seat lock manager.

MILE 02

Wound

If the requester is older than the lock holder, the holder is preempted and rolled back — the older transaction always proceeds.

MILE 03

Wait

If the requester is younger, it queues instead of rolling back anyone — no wasted work, no circular wait.

MILE 04

Commit

The seat lock is granted, the berth is confirmed, and the ticket is issued — deadlock-free by construction.
