Concurrent ticket booking, without the collisions.

Every seat lock resolves through timestamp ordering — wound–wait — so two passengers can never hold the same block section at once. Older requests wound younger ones; younger requests wait their turn.

Wound

An older transaction reclaims the seat from a younger one holding it — the younger request is aborted and must retry.

Wait

A younger transaction requesting a seat held by an older one queues instead of forcing it out — no younger transaction ever preempts an older holder.

Granted

No conflicting hold exists. The lock is issued immediately and the seat moves to the passenger's transaction.
